## 🧠 What is Vault AI?

Vault AI is a **Retrieval-Augmented Generation (RAG)** application that lets you:

- Organise your PDF documents into named **Vaults** (e.g., *Legal*, *Finance*, *Research*).
- Ask natural-language questions and receive answers **grounded in your actual documents**.
- Run the entire AI pipeline **100 % locally** — the language model lives on your machine, not a server.
- Track conversation history, browse past chats, and download source documents — all behind secure JWT-based authentication.

Think of it as a private ChatGPT that only knows what you tell it, and never sends your data anywhere.

---

## ✨ Key Features

| Feature | Description |
|---|---|
| 🗄️ **Multi-Vault Organisation** | Create isolated vaults per topic or project. Each vault maintains its own separate document collection and chat history. |
| 📄 **PDF Ingestion Pipeline** | Upload PDFs and they are automatically parsed, chunked into 800-token segments with 150-token overlap, and embedded into a vector database. |
| 🤖 **Local LLM (Llama 3.2 1B)** | Answers are generated by `Llama-3.2-1B-Instruct-Q4_0.gguf` via GPT4All — a quantised model that runs on CPU with optional GPU (Vulkan/CUDA) acceleration. |
| 🔍 **Semantic Search** | ChromaDB + `all-MiniLM-L6-v2` sentence embeddings power the vector similarity search used to retrieve the most relevant document chunks. |
| 🛡️ **Context Guard** | A keyword-overlap filter prevents the LLM from hallucinating answers when the question is irrelevant to the vault's content. |
| 💬 **Persistent Chat History** | Every conversation is stored in SQLite and can be browsed, filtered, and searched by keyword. |
| 🔐 **JWT Authentication** | Secure registration and login flow with bcrypt password hashing and signed JWT tokens. |
| 📥 **File Download** | Re-download any previously uploaded source PDF directly from the vault. |
| 🗑️ **Vault & File Deletion** | Delete individual files or entire vaults — purges the vector DB collection, SQLite records, and physical files simultaneously. |
| ⚡ **One-Click Demo Launch** | `demo.bat` boots the entire stack in two terminal windows with a single double-click. |

## 🏗️ Architecture Overview

```
┌─────────────────────────────────────────────────────────┐
│                    Browser (React + Vite)                │
│  Login / Register → Dashboard → Vault Selector → Chat   │
└──────────────────────────┬──────────────────────────────┘
                           │  HTTP (localhost:5173 → 8000)
┌──────────────────────────▼──────────────────────────────┐
│               FastAPI Backend  (port 8000)               │
│                                                          │
│  ┌──────────┐  ┌──────────────┐  ┌───────────────────┐  │
│  │ PDF      │  │  Vector DB   │  │  LLM Service      │  │
│  │ Service  │→ │  Service     │→ │  (GPT4All/Llama)  │  │
│  │(pdfplumb)│  │  (ChromaDB)  │  │                   │  │
│  └──────────┘  └──────────────┘  └───────────────────┘  │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │  SQLite Database (users, vaults, chats, messages)  │  │
│  └────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**RAG Query Flow:**
1. User submits a question in a chat session.
2. The query is converted to a vector embedding by `all-MiniLM-L6-v2`.
3. ChromaDB returns the top-5 most semantically similar document chunks.
4. A **Context Guard** checks that the query keywords overlap ≥ 30 % with the retrieved text.
5. The LLM generates an answer using only those chunks as context.
6. The response (with cited sources) is saved to SQLite and returned to the UI.

## ⚡ Option A — One-Click Launch (No Manual Setup)

> **Best for:** Running a demo quickly when the environment is already set up, or if someone has already run Option B once on this machine.

The `demo.bat` script in the project root will start both the backend and frontend in separate terminal windows with a single double-click:

```
Vault_AI/
└── demo.bat   ← Just double-click this file
```

**What it does:**
1. Opens a terminal window and activates the pre-existing Python virtual environment.
2. Starts the FastAPI backend on `http://127.0.0.1:8000`.
3. Opens a second terminal window and runs `npm run dev` for the frontend.
4. The frontend becomes available at `http://localhost:5173` (printed in the second window).

> **Requirement:** The virtual environment (`backend/venv`) and `node_modules` must already exist. If this is a fresh clone, run **Option B** first.

## 🛠️ Option B — Full Manual Installation

Follow these steps if you are setting up the project for the first time on a new machine.

### Step 1 — Clone the Repository

```bash
git clone https://github.com/your-username/Vault_AI.git
cd Vault_AI
```

### Step 2 — Set Up the Python Backend

```bash
# Navigate to the backend directory
cd backend

# Create a Python virtual environment
python -m venv venv

# Activate the virtual environment
# On Windows (PowerShell):
venv\Scripts\activate
# On Windows (Command Prompt):
venv\Scripts\activate.bat
# On macOS / Linux:
source venv/bin/activate

# Install all Python dependencies
pip install -r requirements.txt
```

> After running `pip install -r requirements.txt`, the following packages will be installed:
> `fastapi`, `uvicorn`, `chromadb`, `sentence-transformers`, `pdfplumber`,
> `python-multipart`, `python-jose[cryptography]`, `passlib[bcrypt]`,
> `sqlalchemy`, `aiosqlite`
>
> GPT4All is **not** in `requirements.txt` — it is a heavy dependency pulled in automatically by the model download script below.

### Step 3 — Pre-Download the AI Models

On first run the application needs two AI models downloaded to your machine:

| Model | Size | Purpose |
|---|---|---|
| `all-MiniLM-L6-v2` | ~90 MB | Sentence embeddings for semantic search |
| `Llama-3.2-1B-Instruct-Q4_0.gguf` | ~1 GB | Local language model for answer generation |

Run the helper script to download them now (so the app starts instantly later):

```bash
# Make sure your venv is still active, then run:
python pre_download_models.py
```

You will see a progress bar. Wait for both models to finish before continuing.

### Step 4 — Set Up the Frontend

Open a **new terminal** (keep the backend terminal open):

```bash
# From the project root
cd frontend

# Install all Node.js dependencies
npm install
```

This installs React, Vite, TypeScript, Tailwind, Radix UI, and all other frontend packages listed in `package.json`.

You need two terminals running simultaneously — one for the backend, one for the frontend.

```bash
npm run dev
```

You should see:

## 🔬 How It Works (Under the Hood)

### PDF Ingestion

When you upload a PDF:
1. **`PDFProcessor`** uses `pdfplumber` to extract raw text page-by-page.
2. **`ChunkingService`** splits the text into 800-character chunks with 150-character overlap to preserve context at boundaries.
3. Each chunk is stored in **ChromaDB** with metadata (`source filename`, `page number`).
4. The filename is recorded in **SQLite** to prevent duplicate uploads.
5. The physical PDF is moved to `data/vaults/<vault_name>/`.

### Answering Questions

When you send a message:
1. The query text is vectorised using `all-MiniLM-L6-v2`.
2. ChromaDB returns the **top 5 chunks** with an L2 distance below 1.2.
3. The **Context Guard** checks that ≥ 30 % of meaningful query keywords appear in the retrieved context — this prevents hallucinations when the question is off-topic.
4. The context and question are passed to **Llama 3.2 1B** (via GPT4All), which generates a grounded answer with citations like `[Source: report.pdf, Page: 3]`.
5. The response is saved to the SQLite `chat_messages` table and returned to the UI.

### Authentication

- Passwords are hashed with **bcrypt** via `passlib`.
- On login, a signed **JWT token** is issued (via `python-jose`).
- All protected endpoints use a `Depends(get_current_user)` FastAPI dependency that validates the token on every request.

## 📡 API Reference

The backend exposes a RESTful API on `http://127.0.0.1:8000`. Interactive docs are auto-generated by FastAPI at:

- **Swagger UI:** http://127.0.0.1:8000/docs
- **ReDoc:** http://127.0.0.1:8000/redoc

### Key Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/auth/register` | Register a new user |
| `POST` | `/auth/login` | Log in and receive a JWT |
| `GET` | `/vaults` | List all vaults for the current user |
| `POST` | `/vaults` | Create a new empty vault |
| `DELETE` | `/vaults/{vault_name}` | Delete a vault and all its data |
| `GET` | `/vaults/{vault_name}/files` | List files in a vault |
| `POST` | `/upload` | Upload a PDF to a vault |
| `GET` | `/vaults/{vault_name}/files/{filename}/download` | Download a source PDF |
| `DELETE` | `/vaults/{vault_name}/files/{filename}` | Remove a file from a vault |
| `POST` | `/chat/query` | Send a message and receive an AI response |
| `GET` | `/chats` | List all chat sessions (with optional search) |
| `GET` | `/chats/{chat_id}/messages` | Get all messages in a chat |
| `GET` | `/users/{user_id}` | Get the current user's profile |
| `GET` | `/` | Health check — returns `{"message": "Vault AI Backend is running"}` |

## ⚙️ Configuration

### Backend Ports

The backend and frontend communicate on hard-coded localhost ports. You can change them:

| Setting | File | Default |
|---|---|---|
| Backend port | `demo.bat` and your `uvicorn` command | `8000` |
| Frontend port | `frontend/vite.config.ts` | `5173` |
| API base URL (frontend) | `frontend/src/app/constants.ts` | `http://127.0.0.1:8000` |

### LLM Model

The model is specified in `backend/app/services/llm_service.py`:

```python
# Line 5 — change model_name to swap models
def __init__(self, model_name="Llama-3.2-1B-Instruct-Q4_0.gguf"):
```

Any GGUF model compatible with GPT4All can be used. Browse available models at https://gpt4all.io.

### Chunk Size

Adjust chunking behaviour in `backend/app/main.py`:

```python
# Line 59 — tune chunk_size and chunk_overlap
CHUNK_SERVICE = ChunkingService(chunk_size=800, chunk_overlap=150)
```

## 🩺 Troubleshooting

| Problem | Likely Cause | Fix |
|---|---|---|
| `ModuleNotFoundError` on backend start | venv not activated | Run `venv\Scripts\activate` first |
| `npm: command not found` | Node.js not installed | Install from https://nodejs.org |
| Backend starts but frontend can't connect | Wrong API URL or backend not running | Check `constants.ts` and confirm backend is on port 8000 |
| "PDF contained no extractable text" | Scanned / image-only PDF | Use a PDF with selectable text, or run OCR first |
| LLM model download fails | No internet on first run | Run `python pre_download_models.py` while connected |
| Answers are always "I cannot find the answer..." | Context Guard filtering | Try more specific questions or check if the PDF was indexed correctly |
| `409 Conflict` on upload | File already in vault | The same filename has already been uploaded to this vault |
| Port 8000 or 5173 already in use | Another process occupies the port | Change the port in `demo.bat` / `vite.config.ts`, or kill the conflicting process |
